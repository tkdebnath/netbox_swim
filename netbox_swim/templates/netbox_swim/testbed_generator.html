Quick Start — Genie CLI
0. Load credentials 
source env/swim.env
1. Learn current state (Pre-Check) 
genie learn ospf interface routing vrf \
  --testbed-file testbed.yaml \
  --output pre_snapshot
2. (Perform your changes / upgrade)
3. Learn new state (Post-Check) 
genie learn ospf interface routing vrf \
  --testbed-file testbed.yaml \
  --output post_snapshot
4. Generate Diff Report 
genie diff pre_snapshot post_snapshot \
  --output diff_report

About pyATS Testbeds

A testbed file is a YAML file that defines your network devices — their hostnames, IP addresses, OS type, and credentials. It's the entry point for all Cisco pyATS and Genie operations.

Genie CLI vs Python API
Genie CLI (genie learn) Python API (device.learn())
Output Full directory structure — one JSON per feature per device Python Ops objects in memory — must explicitly serialize
Diff genie diff pre/ post/ — automatic file comparison Must use Diff(pre.info, post.info) manually
Best For Pre/post upgrade verification, quick audits Integration into automation pipelines
